What is Obspy ?

Obspy is a Python library for seismological data processing. It provides many seismological signal processing functions as well as date and time manipulation.

Stream Trace filter read stats ndarray trim dict-like

Basic structure

      The main structure of the Obspy library consists of the Trace and the Stream object. The Stream object contains one or more Trace objects. The Trace is in a sense, a seismic record. Every Trace has the .data property that we can get the data of the record in Numpy ndarray form and the .stats property that points to a dict-like object with name and value pairs of metadata information about the record (eg. station, channel, sampling rate). In addition, a Trace object contains several methods to manipulate the record. An example of these methods include the filtering of the data between a specific frequency range, using the filter() method and the splitting of the data array between a specific start and end time range using the trim() method. The date and time in Obspy are both represented as Obspy UTCDateTime objects that is based on a high precision POSIX timestamp. It is used for handling and representing time and date information in the context of seismology and seismic data analysis.

      The obspy library utilizes miniSEED files in its core, which are binary files used to store time series seismic waveform data. It is a compact and efficient format that includes information about the station, location, timing, and the actual waveform data. MiniSEED files are widely used in seismology and are the standard format for sharing and archiving seismic data. To read an mseed file using the Obspy library apply the read() method. To illustrate this, we implement an example of reading an mseed file, applying some processing to it and saving the Stream to a location.

First let's import the necessary libraries:

Generating Trace and Stream objects from data

      Instead of reading from a seismological file format, you can also create a Trace and/or a stream object using the obspy.core.trace.Trace and obspy.core.stream.Stream classes. Let's remember again, that a stream consists of one or more than one traces. To generate a trace object, start by passing the record data values into the data parameter of obspy.core.trace.Trace as a numpy ndarray and the metadata of the record into the header parameter as a Python dict. Repeat this for as many traces as you need to create and then pass all of them as a Python list, through the obspy.core.stream.Stream class, to generate a stream object. Let's check an example of this.

Suppose we have a seismic record with 3 channels (E, N, Z), recorded in LEF2 station in a text file. Its metadata are located in the first rows of the file and the data of each component, after 9 lines of metadata. We are going to read this file and create a stream object with 3 traces.

First we read the file and save its metadata in a dictionary form and its data into a numpy ndarray form: 

# use the Python open method to open the file and read its metadata
                        with open('20140207-085940-LEF2-data.txt') as fr:
                              # skip the first line by reading it
                              fr.readline()
                              # read the station name
                              station = fr.readline().split(':')[1].strip()
                              # read the starting datetime of the record
                              dt_start = fr.readline().split(':', 1)[1].strip()
                              # read the sampling frequency in Hz
                              fs =  fr.readline().split(':')[1].strip(' Hz\n')
                              # read the number of sample points
                              npts = fr.readline().split(':')[1].strip()
                              # skip 2 lines
                              fr.readline()
                              fr.readline()
                              # read the components
                              compos = fr.readline().split(':')[1].split()
                              # show all the parameters that we read from the header of the file
                              print(station, dt_start, fs, npts, compos)
LEF2 2014-02-07T08:59:40.000000Z 100.0 18100 ['E', 'N', 'Z']

Before we add the metadata into the dictionary, we convert the number of points (npts) into an int datatype, the sampling frequency (fs) into a float datatype and the start time of the record (starttime) into a UTCDateTime object. Also it is important for the keys of the dictionary that we will create, to be one of the options provided in the obspy.core.trace.Stats

At this time we have the metadata and the data of the traces. Using these two parameters, we can create the Trace objects. To do this, loop trough the components, create a header dictionary file for each Trace and add its data: 

# create an empty list to append each trace object
                        lt_traces = []
                        # then create a trace for each component
                        for compo in compos:
                        # create a dictionary object and add there the metadata
                        dict_header = {}
                        dict_header["station"] = station
                        dict_header["npts"] = int(npts)
                        dict_header["sampling_rate"] = float(fs)
                        dict_header["starttime"] = UTCDateTime(dt_start)
                        dict_header["channel"] = compo
                        # create the trace object
                        tr = Trace(data=df_data[compo].to_numpy(), header=dict_header)
                        # append the trace into the traces list
                        lt_traces.append(tr)
                        print(lt_traces)

Finally, crete the Stream object, from the list of the traces:

# create the stream by inserting the list of traces into the obspy.core.stream.Stream class
                    st = Stream(lt_traces)
                    print(st)
