If you have a file that you would like to use with the available tools, this is a convenient conversion tool that allows you to convert data defined in various file formats into the MSEED file format. This tool supports the formats: .xlsx, .csv, .txt, .dat. Start by selecting the appropriate format of your file, fill in the available file parsing parameters (delimiter, number of rows to read, etc.) and then load your data file. After loading the file, check the first five rows that the program reads, at the sample table shown below the file parsing parameters. If it represents correctly you data, continue with the second set of parameters that are related to the sampling process of converting analog signals to digital form (sampling frequency, station name, etc.). Lastly, convert your loaded file to mseed.

As displayed in the table below the file upload parameters after loading the file, the program presents the first five rows of your data for verification. The leftmost column, indicated by bold numbers, represents the index and is automatically assigned an array of numbers from 0 to the total number of rows. This is just a default index created and it won't be included in the conversion. If you file does not contain any header, the column names (the bolded row at the top) are assigned default values from 0 to the total number of columns. However, this is also not gonna be included in the conversion procedure.

Take into account the following about the file parameters:

  • In case of uploading a TXT or DAT file, be careful of the delimiter. Select the delimiter of your columns correctly in order for the program to run. Use the "space" option if your columns are separated by one or more blanks or spaces.
  • Utilize the skiprows entry if you want to skip some rows at the beginning of your file (either format). If you skip more rows than the total rows of your file, the final table will be empty and it will give you an error!
  • Insert a value into the # rows to read field to read a specific number of rows from your file. If the number inserted is greater than the total rows of your file, it is gonna read all the rows.
  • The program can just read three columns in case of two horizontals and one vertical components, or two, in case of one horizontal and one vertical component. For this reason, you need at least two columns in your file. If your file has one or no columns you can't use it.
  • In case you have more than three columns in your file, use the columns to read option to specify two or three columns to insert, using comma separated numbers in case of a .csv, .txt or .dat file, starting from the number one (e.g., 1,2,4 or 2,4 or 1,4) and comma separated column letters (e.g., A,B,C or B,D or A,C,B) in case of an Excel file. You can't upload more!
  • If you file contain headers check the has headers option.
  • if skiprows, # rows to read and has headers options are used, first the skiprows is going to be applied, then the has headers option and lastly the # rows to read. For example, if skiprows=2, # rows to read=10, and has headers=True, then first 2 rows will be skipped, then the first new row after skipping, will be set as the header and then the first 10 rows after the header, will be read.
  • The program does not have a file icon for the .dat file format, but you can use this kind of format at the .txt file format, file parameters

Take into account the following about the seismic parameters:

  • The "starting date" is optional. If no date is selected, a default value of "1970-01-01" will be applied. However, it is recommended to select a date especially in case you intend to select the P & S wave arrivals.
  • As with the "starting date", the "starting time" is also optional but recommended. The default value is "00:00:00".
  • The station name is optional. The station name should contain three to six letters from a-z or A-Z, and/or numbers from 0-9!
  • The "channels" option are obligatory and they need to have one vertical component (Z) and at least one horizontal component (E and/or N). If you provide three columns of data each column corresponds to one vertical (Z) and two horizontals (E and N). If you provide the same channel for a column or if you don't include the vertical component (Z) it will give you an error.
  • Lastly, you need to insert the sampling frequency in Hertz or the sample distance in seconds for the correct x-axis representation.
